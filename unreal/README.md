# EmoBot: Unreal Engine 5 Digital Twin Setup

This folder documents the Unreal Engine 5 MetaHuman and ConvAI integration for the EmoBot system. Because Unreal Engine projects contain large binary assets (~1.5GB of MetaHuman files), we version the **configuration files, project descriptors, and blueprints** instead of the entire binary project.

---

## 1. Project Requirements & Plugins

To reconstruct or run the digital twin, you need an Unreal Engine 5 project (v5.3 or higher recommended) with the following plugins enabled:
* **ConvAI Plugin**: Connects the digital twin character to the ConvAI backend API for natural language conversational responses.
* **JSON Blueprint Utilities** (or a similar JSON helper plugin): Used to parse and read the `input.json` file.
* **Live Link / Live Link Control Rig**: Used for MetaHuman body/facial mapping and animation data.

---

## 2. MetaHuman Setup (Natalia)

1. Import the MetaHuman character **Natalia** from Quixel Bridge into your Unreal project.
2. Open the Natalia Blueprint class (`BP_Natalia`).
3. Ensure the construction script is left as default (it binds body parts, sets master pose components, and configures hair/face material instances).

---

## 3. Custom Event Graph Logic (JSON File Watcher)

The custom logic that powers the emotion bridge lives in the **Event Graph** of `BP_Natalia` (or your Level Blueprint). It polls the `input.json` file generated by the ROS2 Bridge.

### Blueprint Polling Loop Logic:

1. **Timer Setup / Tick**:
   * On `Event BeginPlay`, start a loop using `Set Timer by Event` (e.g., repeating every `0.1` to `0.2` seconds) or execute on `Event Tick` with a delay.
   
2. **File Check**:
   * Use a node like **Load File to String** (or JSON utilities file reader) pointing to the bridge's output path:
     `D:\Unreal_Projects\Final_V2\input.json`
     
3. **Parse JSON**:
   * Use **Get Character Attribute / Parse JSON String** to extract the values from the JSON.
   * Typical JSON structure polled:
     ```json
     {
       "text": "speaking:I am angry, very angry, absolutely furious"
     }
     ```
     
4. **Trigger Speech/Animation**:
   * Detect if the text starts with `speaking:`.
   * Strip the prefix and pass the inner speech text (e.g., `"I am angry, very angry, absolutely furious"`) to the **ConvAI Character Component** using the **Send Text to ConvAI** node.
   * Trigger the corresponding animation or morph target values for the matching emotion.

---

## 4. Recreating the Blueprints

You can find the raw text configurations or Blueprint node code clips in this directory under `/blueprints`. To import them:
1. Open the `.txt` files in a text editor.
2. Copy the entire contents (`Ctrl + A` then `Ctrl + C`).
3. Open your Unreal Engine Blueprint Event Graph.
4. Right-click and press **`Ctrl + V`**. The nodes, connections, and variables will be recreated instantly.
